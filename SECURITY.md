# Security and link safety

This repository is a curated catalog. It does not execute listed projects, validate their security, or endorse them for a particular deployment.

## Report a problem

Use a private GitHub security report when an entry, script, or generated catalog creates a security risk. For an ordinary broken link, archived project, misleading description, or maintenance concern, open an issue.

Do not include:

- live API keys, tokens, passwords, or connection strings;
- unpublished vulnerabilities in a listed third-party project;
- private client or employer material;
- personal or sensitive data.

Report vulnerabilities in a listed third-party project to that project's maintainer. This repository can annotate or remove the entry, but it cannot coordinate disclosure on the upstream maintainer's behalf.

## Use boundary

A successful link check proves only that a URL responded. Review a project's current documentation, license, maintenance status, security policy, data handling, and deployment model before adoption.
